# Word Markdown Converter

## Overview

The project implements a modular pipeline for converting DOCX files into Markdown, splitting the Markdown into structured articles, generating a README index, and cleaning generated artifacts. The system is designed for reproducibility, determinism, and compatibility with Docker-based workflows.

## Docker Configuration

> **Important note about environment configuration**
>
> To run this project correctly, you must select the appropriate `docker-compose` file inside your `setup.sh`.  
> Update the `COMPOSE_FILE` variable according to your platform:
>
> ```bash
> # GPU (AMD ROCm)               ‚Üí docker-compose.gpu.yaml
> # CPU (Intel/Windows/Linux)    ‚Üí docker-compose.cpu.yaml
> # ARM (Apple Silicon M1/M2/M3) ‚Üí docker-compose.arm.yaml
> ```
>
> This ensures that Ollama uses the correct architecture (GPU, CPU, or ARM) during processing.

## convert.py

### Purpose
Converts a DOCX file into Markdown and extracts all H1 headers.

### Behavior
- Reads `/workspace/input/article.docx`.
- Converts it to `/workspace/output/article.md` using Pandoc.
- Extracts all `#` headers from the generated Markdown.
- Writes them to `/workspace/output/headers.log` in numbered format.

### Generated Files
- `article.md`
- `headers.log`

## split.py

### Purpose
Splits the Markdown file into structured articles and generates a README.

### Behavior
- Reads `/workspace/output/article.md`.
- Detects all H1 headers.
- Accepts two mandatory CLI parameters:
  - `--in-readme`: list or range of section numbers to include in README.
  - `--articles`: list or range of section numbers to convert into article files.
- Generates Markdown files for selected article sections.
- Builds a README containing:
  - Title
  - Selected sections
  - Table of generated articles
- Logs all operations to `/workspace/output/split.log`.

### Generated Files
- `articles/*.md`
- `README.md`
- `split.log`

## purge_output.py

### Purpose
Cleans only the files generated by the pipeline, preserving the folder structure.

### Behavior
Deletes:
- `article.md`
- `headers.log`
- `README.md`
- `split.log`
- Entire `articles/` directory

Does **not** delete:
- `.gitkeep`
- The `output/` directory itself
- Any user-created files

## Workflow Summary

1. Run `convert.py` to generate Markdown and header logs.  
2. Run `split.py` with explicit section selections to generate articles and README.  
3. Run `purge_output.py` to clean generated artifacts.

## Design Principles

- Deterministic output  
- Explicit section selection via CLI  
- Safe cleaning that preserves repository structure  
- Compatibility with Docker Compose and mounted volumes  
- Avoidance of accidental deletion of user files
